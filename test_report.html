Redis açık.
🔧 Öneri: Kısa öneri: Redis parola koy.

📋 Sorun
Redis sunucusu parolasız erişilebilir.
⚠️ Neden Önemli
Redis auth'suz açıksa saldırgan tüm veriyi okuyabilir, silebilir. Daha kötüsü: `CONFIG SET dir /home/redis/.ssh` + `CONFIG SET dbfilename authorized_keys` + `SET x "ssh-rsa..."` + `SAVE` kombinasyonuyla sunucuya SSH anahtar yazıp tam sistem kontrolü alabilir. 2017'den beri internet'e açık Redis'lerin binlercesi ransomware'e kurban oldu.
🔧 Nasıl Düzeltirim
1. Güçlü parola + localhost bağlama
`/etc/redis/redis.conf` düzenleyip Redis'i yeniden başlat:
# Sadece localhost'tan erişim
bind 127.0.0.1 ::1
# Güçlü parola — 32+ karakter, rastgele
requirepass "<64 karakter rastgele dizi>"
# Protected mode aktif kalsın
protected-mode yes
# Tehlikeli komutları devre dışı bırak
rename-command FLUSHDB ""
rename-command FLUSHALL ""
rename-command CONFIG ""
2. ACL (Redis 6+) kullanımı
Modern yaklaşım — her uygulama için ayrı kullanıcı:
ACL SETUSER myapp on >guclu_parola ~myapp:* +@read +@write -@dangerous
3. Güvenlik duvarı
Uzak Redis gerekiyorsa (bulut) sadece uygulama IP'lerinden:
sudo ufw allow from <app_ip> to any port 6379
sudo ufw deny 6379
4. TLS
Redis 6+ TLS destekler — production'da şart:
tls-port 6380
tls-cert-file /path/to/cert.pem
tls-key-file /path/to/key.pem
✅ Doğrulama
`redis-cli -h <ip> PING` parolasız CHANNELS error dönmeli (NOAUTH). Parolayla `redis-cli -a <parola> PING` → PONG.

Orta CSP eksik https://test.example
Content-Security-Policy yok.
🔧 Öneri: Kısa öneri: CSP ekleyin.
📘 Detaylı Onarım Rehberi — göstermek için tıklayın
📋 Sorun
Content-Security-Policy (CSP) header'ı yanıtta yok.
⚠️ Neden Önemli
CSP, tarayıcı seviyesinde XSS ve data injection saldırılarına karşı en etkili savunmadır. Header yoksa saldırgan injection yaparsa tarayıcı hiç ayrım yapmadan gelen her script'i çalıştırır. Saldırı senaryosu: bir formda XSS açığı varsa, saldırganın kullanıcıya tıklattığı link ile cookie/session çalınır. CSP bunu engelleyebilir.
🔧 Nasıl Düzeltirim
Nginx
`nginx.conf` veya ilgili site dosyasında `server` bloğuna ekleyin. Önce Report-Only ile başlayın, hataları izleyip gerçeğe geçin:
add_header Content-Security-Policy-Report-Only "default-src 'self'; script-src 'self'; style-src 'self' 'unsafe-inline'; img-src 'self' data:; report-uri /csp-report" always;
Apache
`httpd.conf` veya `.htaccess` dosyasına ekleyin:
Header always set Content-Security-Policy-Report-Only "default-src 'self'; script-src 'self'; style-src 'self' 'unsafe-inline'"
IIS (web.config)
`<system.webServer><httpProtocol><customHeaders>` altına ekleyin:
<add name="Content-Security-Policy" value="default-src 'self'; script-src 'self'; style-src 'self' 'unsafe-inline'" />
Cloudflare Dashboard
Rules → Transform Rules → Modify Response Header → `Content-Security-Policy` set et. Bu CDN seviyesinde uygulanır, sunucu yapılandırmasına dokunmadan kurulabilir.
✅ Doğrulama
Düzeltmeyi doğrulayın: `curl -I https://siteniz.com | grep -i content-security-policy` çıktısında header görünmeli. Tarayıcıda DevTools → Network → ilgili isteği seç → Response Headers.

Orta HSTS eksik https://test.example
HSTS yok.
🔧 Öneri: Kısa öneri: HSTS ekleyin.
📘 Detaylı Onarım Rehberi — göstermek için tıklayın
📋 Sorun
Strict-Transport-Security (HSTS) header'ı yok.
⚠️ Neden Önemli
HSTS yoksa tarayıcı, ilk ziyarette HTTPS yerine HTTP'ye bağlanmaya çalışabilir. Ağ üzerindeki saldırgan (kafe wifi'si, yolcu) bu ilk HTTP isteğini yakalayıp SSL stripping saldırısı yapabilir — kullanıcı sahte bir HTTP sitesi görür, parolasını girer, saldırgan okur. HSTS tarayıcıya 'bu domain'de HER ZAMAN HTTPS kullan' diye söyler.
🔧 Nasıl Düzeltirim
Nginx
Sadece HTTPS server bloğunda, includeSubDomains + preload için:
add_header Strict-Transport-Security "max-age=31536000; includeSubDomains; preload" always;
Apache
`httpd.conf` içinde `<VirtualHost *:443>` altına:
Header always set Strict-Transport-Security "max-age=31536000; includeSubDomains; preload"
IIS (web.config)
HTTPS binding'i olan site için customHeaders altına:
<add name="Strict-Transport-Security" value="max-age=31536000; includeSubDomains; preload" />
Cloudflare
SSL/TLS → Edge Certificates → HSTS → 'Enable HSTS'. Max Age: 12 months, Include Subdomains: on, Preload: on. Cloudflare otomatik uygular.
✅ Doğrulama
`curl -I https://siteniz.com | grep -i strict-transport` çıktısında `max-age=31536000` görünmeli. HSTS preload listesine başvurmak için: https://hstspreload.org
